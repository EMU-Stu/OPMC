# EMU-Stu 开源与协作指南

本指南由 EMU-Stu 开源项目委员会（OPMC）制定，旨在规范学校各实验室之间的代码协作、项目管理及技术沉淀。本组织采用“Owner 顶层合规、项目完全自治、负责人全权管理”的去中心化架构，给予各课题组最大程度的研发自由。

## 一、 项目生命周期

项目从构思到成熟分为五个阶段：筹划 ➔ 立项 ➔ 开发（内测） ➔ 沉淀（公测） ➔ 归档。

## 二、 筹划与立项

1. **课题分类**

   公开课题：旨在全校或全网开源的项目，鼓励多实验室联合开发。

   私有课题：涉及实验室核心技术、国家竞赛或未发表论文的研究，需严格控制可见性。

2. **立项与迁移流程**

   初始创建（0% - 50%）：项目负责人需先在个人 GitHub 账号下创建仓库并进行基础开发。组织内不提供“从零孵化”的占位仓。

   申请迁移（50%）：当项目完成度达到计划的 50%（具备核心功能或基本框架）时，在 [EMU-Stu/OPMC](https://github.com/EMU-Stu/OPMC) 管理仓提交 Issue 进行立项申请。

   仓库迁移：

   公开项目：Owner 审核通过后，负责人将个人仓库迁移（Transfer）至 EMU-Stu 组织，并将可见性保持为 Public。

   私有项目：Owner 审核通过后，负责人将个人仓库迁移（Transfer）至 EMU-Stu 组织。注：由于组织后台限制了普通成员建仓，迁移后由 Owner 协助确认可见性锁死为 Private。

3. **权限分配（项目自治）**

   仓库成功进入组织后，执行以下权限映射，实现真正的项目自治：

   项目负责人：自动获得/保留该仓库的 Admin (管理员) 权限。作为该项目的“第一最高负责人”，拥有该仓库的完全控制权。

   团队及审核人管理：负责人可根据项目需求，在仓库的 Settings -> Collaborators 中自主添加组员（或外部合作者）并分配权限，亦可自主指定其他核心成员共同担任代码审核人（Reviewer）。Owner 不干涉该仓库内部的人员任免。

## 三、 协作与代码规范（核心：PR 模式）

为防止代码冗余和“乱改”现象，各项目需强制执行 Pull Request 协作模式。

1. **协作权限逻辑**

   公开项目：全员及全网拥有 Read 权限。非核心贡献者必须通过 Fork 模式 参与。

   私有项目：仅限该项目负责人自行邀请的“核心负责人”与“外部合作者”可见。

2. **提交与审核规范**

   **建议各项目负责人自行在仓库设置中开启分支保护，不得直接 Push 代码至主分支。**

   开分支：团队成员在仓内创建 feat/xxx 或 fix/xxx 功能分支（或通过个人 Fork 仓开发）。

   发起 PR：代码编写完成后，向原仓库的 main 分支发起 Pull Request。

3. **代码审核**

   所有 PR 必须经过项目负责人（Admin）或其指定的审核人的 Approve。

   负责人需对项目的整体代码质量、注释规范、以及是否存在敏感数据泄露（如服务器私钥、本地绝对路径等）负总责。

4. **合码**

   审核通过后，由负责人或获授权的审核人点击 Merge 合并代码。

## 四、 实验室隔离与安全策略

为了在给予最大自治权的同时保障组织整体安全，执行以下硬性规则：

基础权限屏蔽：组织的 Base permissions 统一设置为 None。未被明确邀请的成员，进组也看不到任何非公开代码。

禁止跨权自建：普通成员无权在组织内直接创建新仓库。所有新项目必须走个人开发 ➔ 申请迁移的路径，以确保组织内的每一个仓库都有明确的 Admin 承担资产安全责任。

动态清退义务：鉴于负责人拥有 Admin 权限，在组员毕业或离开课题组时，负责人有义务及时在仓库设置中移除该组员，一键收回其访问权限。

## 五、 项目阶段标注

为了清晰标识项目状态，README 标题后需注明状态标签：

- **实验性**：早期开发阶段，代码可能频繁重构。

- **稳定版**：已通过内测，功能健全，可供其他实验室参考学习。

- **归档**：项目已结项或论文已发表，进入只读维护状态。

## 六、 行为准则

- **尊重知识产权**：引用他人开源代码需严格注明出处。

- **严禁泄密**：不得将包含服务器密钥、内网 IP、私密数据集推送到公开仓库。

- **交付义务**：项目归档前，负责人需确保 README 包含详尽的部署指南（如环境依赖、硬件引脚连接图、启动命令等），确保学弟学妹们可复现。

_致项目负责人：如果你准备好了迁移你的第一个项目，请立即在 [OPMC](https://github.com/EMU-Stu/OPMC) 仓库按上述流程发起 Issue 立项申请。成功迁移后，你将作为该仓库的 Admin 全权主导后续的开发与团队管理。_
